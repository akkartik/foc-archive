# Miles Sabin 2019-01-03 11:26
I'm in the typed functional programming camp. I've been working with Scala since the beginning in 2004. I'm a member of the Scala SIP (ie. language design) committee and a contributor to both Scala 2 and the upcoming Scala 3 (aka Dotty) compiler ... currently my day job is 50% work on the latter. I'm also the author of shapeless, Scala's main generic programming library, and a cofounder of Typelevel, an umbrella group for Scala projects which focuses on the typed functional end of the Scala spectrum combined with a desire to create a welcoming environment for people coming from outside that tradition.
Types are very much my thing, and I take the view that dependent types in particular are the future. I think of Scala and Haskell as being the partially dependent status quo, and languages like Agda and Idris showing the way ahead.

# Miles Sabin 2019-01-04 02:59
Incremental compilation is joint effort between the compiler (this is pretty much the same for scalac and dotc) and build tools. These days most build tools share a common library called Zinc to handle their end of it: https://github.com/sbt/zinc.
# Miles Sabin 2019-01-04 03:01
There's no real incrementality from the compilers PoV. It will only ever handle complete compilation units, taking them from source, translating them relative to existing compilation artefacts and producing compilation artefacts.
# Miles Sabin 2019-01-04 03:02
The key to incrementality is what counts as an existing compilation artefact.
# Miles Sabin 2019-01-04 03:05
Build tools based on zinc keep track of the relationship between sources and compilation artefacts. They insert a phase into the compiler pipeline which allows them to accurately track semantic dependencies between sources and artefacts and then keep track of local source changes which require recompilation to be propagated more widely.
