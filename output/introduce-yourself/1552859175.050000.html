# Edward de Jong 2019-03-18 21:35
A super loosely typed language like JS is going to create problems for the iDE. Since it was designed behavior that you can do crazy stuff like 1+ "21" and (is it 22 or 121 or "121")?. The whole JS/CSS/HTML stack is fatally flawed. At no time in history did you mix 3 languages together in one source file, constantly switching back and forth between differing syntaxes, which don't even agree on how comments are formed (JS comments not equal to CSS comments). Why does CSS that can set a style have no IF statement or arithmetic, and why does JS which can do arithmetic and has IF can't draw? A horrible mess and i don't care if it is #1 on the planet, it is still a horrible mess.
