# Doug Moen 2019-05-25 02:23
When I was a student, I read a number of classic texts on programming language design:
  • Peter Landin's 1964 paper on ISWIM, "The Next 700 Programming Languages",
  • Tennant's "Principles of Programming Languages" (the 1970's),
  • a PhD thesis, "Abstraction Mechanisms and Programming Language Design" (the 1980's).

    They all say the same thing. To the greatest extent possible, you should be able to replace any node in an abstract syntax tree with a variable reference, or a function call, or a block (aka let-expression). Privately, I call this "the principle of abstraction".

    This leads to things like the Lisp macro system, and staged compilation, and partial evaluation, and dynamic typing.
