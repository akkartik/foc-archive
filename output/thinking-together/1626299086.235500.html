# Jack Rusher 2021-07-15 00:53
The literature* seems to indicate that the strongest correlation between in-born attribute and high levels of programming ability is working memory. This might suggest that reducing the need for working memory would be one of the best ways to make programming easier for those not "born to it."

(* This paper and its references would be a good place to start if you're interested in this sort of thing.)

# Ray Imber 2021-07-25 19:01
That quote reminds me of the memory palace / method of loci technique used in speed memorization contests. :nerd_face:
Ryan Singer also talks about this idea of Multi-scale networks to solve problems (though in a slightly different context)

I think we already do something like this in programming today:
A for loop is an abstraction for a set of assembly code with an increment operation, a comparison operation, and a jump operation.
A function or method is an abstraction for a set of logically related instructions.
A module or API is a an abstraction that represents a set of related functions.
edit: Additionally, a DSL or a macro is somewhere between a function and an API. I haven't decided which yet. :stuck_out_tongue:
