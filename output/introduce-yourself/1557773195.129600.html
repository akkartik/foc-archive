# Ian Bicking 2019-05-13 11:46
Hi! I’m Ian (but you could tell that already), from Minneapolis.

I’m particularly interested in programming environments – for instance, notebook interfaces seem like the most notable programming exploration of the moment. But the specific project I’ve been mulling over and experimenting with is using fully automated testing (like fuzzing) of interactive applications to collect execution information to support the programmer. I imagine a system where you can get a map of what the application actually does (intentional or not), and then you can explore down into why it does those things.

But it’s a side project, so maybe more dream than reality, but I’m sure I’m not alone :wink:

# Ian Bicking 2019-05-14 09:24
It doesn’t exist in public at all at this point, I’m still hoping to hustle up something at work in the area.

All my experimentation has been in the area of web applications – the browser interface is well-defined and sandboxed, so you can really tell what can and cannot cause changes in an application (e.g., which elements have event listeners). Browser debugging interfaces also offer a well-documented way to inspect relatively high-level behavior.

I’ve been able to generate random runs in real-world applications, but I haven’t convinced myself that I can actually thoroughly explore an application automatically. And then generating insight from those runs is also something I haven’t managed to do. So it’s pretty early-stage.
