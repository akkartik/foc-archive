Another thought on this topic:
There is a hierarchy of code interaction with decreasing frequency:

1. Reading
2. Patches (e.g. change a string)
3. Minor changes (e.g. write a function)
4. Major changes (e.g. restructure code)
5. Create a new project

And I think code editors should be optimized for this interaction pattern.
