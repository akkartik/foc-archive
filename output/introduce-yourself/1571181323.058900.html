# Eric Hanson 2019-10-15 16:15
Hi folks, I'm a long time fan of Edwards and Victor, and the guy behind the aquameta project ( http://aquameta.org ), trying to prove the concept of (for lack of a better name) "datafication" of the stack. Aquameta reimplements the layers of a traditional web stack as relational data, so that rows instead of files are the atomic unit. As an industry, we are really good at building user interfaces to data, and if the whole stack is data, UI that manipulates it very easy to develop. Explorinng concepts around version control of relational data, datafication of EBNF grammars for data-driven code manipulation from a UI, semantic decoration of the database schema to couple UI components with columns for auto-crud generation that can be progressively enhanced, an on-screen in-app widget editor so that you can edit a UI from within the UI, etc. Demo videos: https://www.youtube.com/watch?v=ZOpj8lvNJtg&t=4s and https://www.youtube.com/watch?v=r8SYrTmUXgU

# Eric Hanson 2019-10-16 02:51
So don't disagree that UX is not great, but this is a key point, that the UI really was just thrown together, and only like 5% of the focus of the project has gone towards UI. What's interesting (at least supposed to be) is that the UI is just editing an underlying relational data model, which can support unlimited different co-existing approaches at UX. All programming happens under the hood as some form of data manipulation. This is just the first barely functional UI, still in the "holy cow it works" stages.
