# Tudor Girba 2019-02-11 23:53
Pharo is a mature language and a strong runtime (one of the fastest VM in the dynamic language world). The Pharo VM uses only one CPU, so the way you deal with distribution is to have one worker image per CPU, not unlike JS, Python or Ruby.
# Tudor Girba 2019-02-11 23:57
@Jack Glamorous Toolkit is a graphical and tooling environment. A Pharo system is made of a VM and image. An image offers the language and runtime, and the environment. Glamorous Toolkit offers an alternative environment, which is half of the image, and arguably, what makes a Smalltalk system special.

In fact, the Pharo distribution already ships the first generation of the Glamorous Toolkit since four years. The new generation that you see at gtoolkit.com is based on a complete new implementation on top of a new graphical stack that has some unique properties and includes a highly flexible and dynamic text editor.
