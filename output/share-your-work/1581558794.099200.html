# Raathi 2020-02-13 13:09
You are right. Currently the code bubbles live in it’s own tab because, like you already pointed out, there is no support for floating tabs yet in VSCode.

I mostly use it to navigate and read code and for that purpose it’s working out well that I initially expected.

However the internal editor JS Library is the Monaco Editor, the same editor which powers the main VSCode editing experience but inside bubbles you would not get your editor preferences or plugins at the moment.
