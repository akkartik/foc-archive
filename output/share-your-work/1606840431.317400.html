# Chris Granger 2020-12-01 09:24
There are a couple of cool things happening here.
1. We’re seeing declarative statements about the world “buttons are flammable.” We get to define concepts in the world and then apply them to things.
2. There are contextual statements. We make a button and at the end say “show the count of clicks on it”
3. The schematic has multiple steps here and slows down replay so you can see the whole thing execute correctly.
4. This is the first glimpse of part nodes in the schematic and that parts can be marked as optional (which is why “on fire” has that toggle)
5. This shows relationship wires (as attached to parts) and signal wires (as attached to processes). Signal wires “fire” when they’re activated.
6. This process doesn’t just update a property value, it changes which parts are active
7. By the end we’ve seen multiple types of logic compose into a single object without any effort.
