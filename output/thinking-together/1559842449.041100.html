# Konrad Hinsen 2019-06-26 07:21
One way to build Wikis with GToolkit with no extra code is to make each Wiki page a Pharo class, with the text in the class comment. Then use class annotations as links. Since the pages are then considered to be source code by Pharo tools, you get version control (via Iceberg, Pharo's git interface) for free. Much of the GToolkit documentation is actually written this way, so just get GToolkit and look at the tutorials to see how this works in practice.

I have been playing with computational wikis in Pharo/GToolkit recently (see https://github.com/activepapers/activepapers-pharo for the details). That took some extra code because I wanted groups of Wiki pages with a shared code basis. One Pharo class is then a package of code, data, and Wiki pages.
