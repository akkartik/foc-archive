# Ivan Reese 2021-08-24 09:58
@Chris Granger — Not a bad thought. Perhaps, if you had more control over shape tweens — say, richer ways to add hints or control how the intermediate motion unfolded. Not to mention that if your source and dest shapes had different vertex counts, you'd get a big explosion of self-intersecting garbage because the vertex numbering was very... unstable. Shape tweens were so unreliable that we ended up doing all our fancy curve stuff using ActionScript and the canvas-like APIs. At which point, Unity + a decent vector shape library (like the one Freya made) is probably no worse than Flash in practice. But in theory.. yeah, something Flash-like with a little more forgiving / robust toolset could be a nice improvement over Unity. You're still in a compile->run workflow, but that's mostly orthogonal.
