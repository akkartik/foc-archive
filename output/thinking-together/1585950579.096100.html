# Edward de Jong 2020-04-03 22:54
The "gotcha" with structured editing is that the minute you propose a graphical interface for data structures, code, screen layouts, etc., you being a project that is a large amount of work, and at 50% complete the system is fairly worthless. The simplicity of text is that a very rudimentary editor gives you a complete system. In the graphical space, you will inevitably need to offer color pickers, gradient designers, layout tools, schema designers, etc., all of which add up.

No question it can be done. MPS is bewildering IMHO.
# ikrima 2020-04-03 23:06
one approach i'm taking is not going full structured editor. The program is still stored in text form and editing happens in text form and the editor is on a constant loop of:
parse AST -> if successful, update stored structure -> update view widgets
on save -> persist both textual and structural data
