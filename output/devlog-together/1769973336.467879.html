# Jimmy Miller 2026-02-01 11:15
Here is a game I made in my language beagle. It is using c interop with raylib for all the drawing. But you'll notice one cool thing, the pause and rewind. What is particularly cool to me about this feature is that the game wasn't designed with that in mind. Instead, on each frame I capture a delimited continuation and store them in a ring buffer. Now we can rewind through them and resume execution at any point in the program.

This is a small step towards some of the things in the https://www.youtube.com/watch?v=72y2EC5fkcE They seem to not be doing this exact strategy. I've considered reaching out to see if I can learn of the details of what they have going on. But as a flesh out more of the robust live coding stuff I want, I definitely think I could be a good amount of the way there.

I also think without too much trouble, I could get to the bret victor style seeing the future. If the game has a simulation step separate from rendering, it should be possible to do that without too much work from the game.
