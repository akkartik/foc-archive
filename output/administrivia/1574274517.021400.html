# Doug Moen 2019-11-21 10:29
Threads are unneeded complexity--they actually get in the way--for real time chat, exchanging timely information. Eg, coordinating the virtual symposium that just ended, over in #research-center. For this use case, archiving old posts is not important.

Threads are useful for intellectual discussions, where you might want to go back and review a discussion from 6 months ago, without having to filter out all of the other unrelated chatter that happened at the same time the discussion was taking place.

Since our motivation for replacing Slack is that we need full historical access to the archive, it follows that the replacement needs to support threads. A corollary is that the Matrix integration should only be used for reading the forum, or for real-time chat. Please use the threaded interface to participate in intellectual discussions.

# J. Ryan Stinnett 2019-11-28 07:45
Permalinks to comments in Riot (at least via three dots -> share message on web client) should produce a https://matrix.to link that allows the recipient to choose a client to view it in. When clicking matrix.to links inside Riot, it will follow them directly and skip the indirection layer. As Jamie mentioned, a proper URI scheme is in the works, which should make this a bit easier.
# J. Ryan Stinnett 2019-11-28 07:49
It's meant to be the neutral, client-agnostic permalink that I think @Kartik Agaram wants. Once there's a proper URI scheme in place, whatever client you prefer should pick up and handle the links directly.
# Kartik Agaram 2019-11-28 07:53
Yeah that matches the format I see. But it starts with https:, so I don't follow why it's a URI 'scheme' that needs a 'client'. My browser is my client for https links, and it seems pretty agnostic. Some wheel is being reinvented here.
# J. Ryan Stinnett 2019-11-28 07:58
Hmm, maybe I am not explaining it well... The current thing at https://matrix.to is not a URI scheme, it's stop-gap temporary way to link without enforcing a single client on the link recipient, who might use any web / mobile / desktop client of their choice to actually view it. The future path is to reserve a proper URI scheme like matrix: once the spec for such URIs is finalised.
