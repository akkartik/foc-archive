# Brian Hempel 2018-12-13 06:43
But it depends on the version of SnS whether that error matters or not. There’s two branches of development at the moment. “Bidirectional” SnS, which is described in the SPLASH 2018 paper, doesn’t use the external solver so the error above doesn’t matter. The “v0.7.1" link is this “Bidirectional” SnS. “Bidirectional” SnS is better formalized and can operate on larger programs but can currently only perform “small” program changes (i.e. modification of literal strings, numbers, or lists). The other SnS branch, call it “Output-Directed” SnS, can perform larger program transformations and is what was demoed at the LIVE workshop (https://www.youtube.com/watch?v=9JcKnJyufD8). It’s available through the “icfp-2018-svg” link. Let me know if you have any questions.
