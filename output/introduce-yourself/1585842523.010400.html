# David Piepgrass 2020-04-03 07:20
"Where do people spend the most of their time in writing software?" In my job, seems like (1) reading code (including 3rd-party stuff with poor documentation), (2) debugging (including reading StackOverflow about error messages) (3) thinking about how to solve problems under the constraint of minimizing "risk" and changes to the zillion lines of code that are already written, (4) writing code to solve the problem, (5) refactoring code, (6) being distracted by more interesting things on the internet, (7) writing test code, (8) commit management - reviewing, writing commit messages, merging, rebasing, fiddling with task board, etc., (9) scrums/meetings, (10) searching for & reading documentation. (Disclaimer: just guessing, and reading code only comes first in low-quality codebases)
