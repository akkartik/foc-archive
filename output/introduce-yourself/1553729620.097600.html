# Wouter van Oortmerssen 2019-03-27 16:33
Hello, World! I'm Wouter van Oortmerssen and I've made a few programming languages (http://strlen.com/programming-languages/) probably the most futuristic being Aardappel (graphical distributed tree rewriting by example, http://strlen.com/aardappel-language/) and Restructor (fully automatic whole program refactoring, also graphical, http://strlen.com/restructor/). # Wouter van Oortmerssen 2019-03-28 20:41
Yup, you have a pretty good picture of the type system. The goal is Python like comfort and productivity with a C# or even Rust-like safety net :slightly_smiling_face:
# Wouter van Oortmerssen 2019-03-28 20:42
The magic in the type system is that it does flow sensitive specialization, so the type checking is relative to the compile-time call-graph, as if it evaluates each possible path for every type combination
# Wouter van Oortmerssen 2019-03-28 21:23
One area that changed recently (and I yet have to document) is memory management: I implemented a lifetime/borrow checker similar to Rust, but unlike Rust it is mostly transparent to the programmer. In the base case it results in compile-time reference counting. Also it gained "zero-cost abstraction" in the sense that it has inline structs now
# Wouter van Oortmerssen 2019-03-28 21:25
Also the beginnings of concurrency, in fully distributed, coordinated tuple space fashion
