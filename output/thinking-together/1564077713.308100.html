Here's a quote from webassembly.org

> WebAssembly modules will be able to call into and out of the JavaScript context and access browser functionality through the same Web APIs accessible from JavaScript.

I should have said "Web APIs" instead of "JS APIs". Basically, the APIs that expose browser functionality to your code. Those APIs are designed around JS semantics, so there's probably no escaping some dissonance if you're trying to run a very pure, safe language inside WASM but still actually make something that people can use.

(Yes, this is not that different from running a pure, safe language on top of any other platform designed for C or POSIX or what have you.)
