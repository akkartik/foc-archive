# Noel 2024-07-07 07:02
All I know about robust computing comes from the Future of Coding episode on it, so this response is probably steeped in ignorance. One thing not mentioned in the episode is probabilistic algorithms. The usual setup is to assume a deterministic computer, and then introduce imperfect information (often via a hash function) giving a small probability of error but achieving a large reduction in time or space requirements. Perhaps the most used algorithm is [HyperLogLog](https://en.wikipedia.org/wiki/HyperLogLog), but I think [MinHash](https://en.wikipedia.org/wiki/MinHash) is particularly easy to understand. This isn't the same as assuming any part of the algorithm can fail, but it might provide a pathway to that model.
