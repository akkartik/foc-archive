# Don Abrams 2019-11-05 07:29
You specify the relationships via code (which creates a code graph rather than a tree).
Then, you specify a starting point that takes expression(s) and it will iteratively reduce it (collapsing nodes in the graph along the way) to it's maximal reduction (via typed lambda calculus)
The most reduced form shows what constraints are free (usually bounded by types so caught at compile time)
