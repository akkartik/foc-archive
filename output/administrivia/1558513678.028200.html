# Konrad Hinsen 2019-05-28 09:28
As for building our own stuff, I am a bit skeptic that anything will ever come out of it, other than a lengthy discussion of what the ideal system should be like. That's the eternal weakness of idealist groups. So... how about a compromise: start with something existing and workable and incrementally add to it. That would (1) ensure there is a working system at any time, (2) ensure that specific developments remain focused, and (3) create an excellent exercise for the real-life fact that every new technology requires a path for transitioning to it.
As an example, we could start from an existing Wiki-type system (which might be just GitHub) to accompany Slack, and then transition to a homebuilt discussion forum that better integrates with the Wiki.

# Doug Moen 2019-05-28 14:41
Benefits of Gitter+Github:
  • single signon to git repos, wikis and chat rooms (using github credentials)
  • github flavoured markdown, with @user and #issue refs, available everywhere
  • chat archives are visible on internet without signon
  • full chat history is archived, searchable and visible
  • Gitter is owned by Gitlab, and is in the process of transitioning to Open Source. Once this is done and Gitter has a Gitlab integration, we could transition to a fully open source system, which could be self hosted and customized, but need not be.
