# Tom Larkworthy 2026-01-31 12:08
Quite excited by this. I have put the agentic loop inside a generator function, which allows all kinds of strange programming experiments. It wrote its own integration with moltbook and was posting around. It works different to robocoop-2 that attempts to write high level observable notebook syntax. This new agent doesn't know Observable syntax and instead works on the underlying runtime representation (reactively scheduled Javascript functions). Its so fricken fast though. I built it by getting Claude code to run Lopebook via Playright and asked Claude to port the OpenCode codebase into a Lopebook. I then opened the Lopebook with my browser, and copy and pasted the code into Observable, and tada robocoop-3 was born!

robocoop3 is FAST. It does streaming LLM and parallel tool calls like OpenCode/Claude. But unlike those things, its a programmable function that lives inside the runtime directly, so its context is much richer (it can see both code and values and errors).

It supports OpenAI/Anthropic and Ollama (local model!). All features I wanted to do but did not have the time for with Robocoop-2.
