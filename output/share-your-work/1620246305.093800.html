# Yousef El-Dardiry 2021-05-16 01:02
Thanks @nicolas decoster!! In most cases it’s fairly similar to Vue / MobX.

The main innovation is the concept of an “implicit observer”. This means that we can store the “observing” context on the actual reactive / observable variable.

e.g.: const store = reactive({}, Observer) Assumes that all reads on store should be registered on Observer. .

This is a nifty trick on which we can build some things like:
• Depend on observable state in React without having to wrap all code / components with observe(() => {..}) like you’d have to do in MobX. So we now only need to use the useReactive hook (source @ https://github.com/YousefED/reactive/blob/main/packages/reac…)
• We could make observables work in async code. Other libraries can’t automatically track whatever observables you use after an await statement. (example @ https://github.com/YousefED/reactive/blob/39cee3d42a29696fa9…)
