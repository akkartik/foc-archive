# Pandi Lin 2026-02-01 10:21
Hi everyone! I’m recently working on Lain, a programming language for distributed computation that attempts to implements some idea from smalltalk/pharo and scheme in propagator model,
The core premise is that the compiler itself is a propagator network: it compiles ASTs into meta-compiler networks where the AST can be partially evaluated at any point. This allows the environment to remain fluid even in distributed settings. I’ve recently synchronized the repository with Gun.js. This allows for a single, universal environment shared across multiple peers, The prototype is in a very early research stage, and the design is evolving rapidly. I’d love to hear your thoughts! Repo: https://github.com/Semi-0/lain-lang

# Pandi Lin 2026-02-03 03:12
A propagator network is different from a usual dataflow network mainly in
what lives at the nodes.
In a propagator network (in the sense of Gerald Jay Sussman’s work),
nodes are autonomous cells that accumulate partial information, rather
than variables that are assigned single values.
Dependencies are implicit in the propagation structure, and computation
proceeds by information growth until a fixed point is reached.

# Pandi Lin 2026-02-03 03:48
it support cyclic connections. A propagator network is evaluated as a fixed-point process:
a propagator runs when a cell’s knowledge is refined (i.e. strictly grows), not merely when some message arrives.

There is no single global evaluation order.
Execution proceeds by information growth until quiescence (a fixed point)
is reached.
