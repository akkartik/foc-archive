# Carlo Kok 2019-05-05 21:38
Standard libraries: We have an stdlib for Swift (where the Dictionary and List are value types and are based on the target frameworks List / Dictionary, so System.Collections.Generic.List<T> on .NET, java.lang.ArrayList<T> on Java, NSMutableArray<T> on Cocoa, etc) , for Go we have a "Slice" type that is convertable to a regular platform array. In the same project you can mix and match languages yes, and trivlally call between things as if they were written in the same language.
