> 3. Show that p ≤ q in P if and only if ↑(q) ⊆ ↑(p).

Part (2) is the forward direction. For the reverse, suppose ↑(q) ⊆ ↑(p). By reflexivity q ≤ q. This means q ∈ ↑(q). So q ∈ ↑(p). This means p ≤ q.

What I like best about this is that there is nothing tricky. We also learn that a monotone map existing corresponds to a kind conditional "if ... then ..." proposition.
# William Taysom 2019-09-25 17:48 #
>> 2. Show that this construction defines a monotone map ↑: op(P) → U(P).

More definition unwrapping. U(P) is the set of upper sets in P ordered by set inclusion ⊆. For ↑ to be a monotone map, ↑ needs to firstly take every element of P to an upper set in P – we showed that in part (1) – and secondly if q ≥ p then ↑(q) ⊆↑(p). Let's prove it.

Take q ≥ p. This means p ≤ q. Consider r ∈ ↑(q). This means q ≤ r. By transitivity p ≤ r. This means r ∈ ↑(p). And so ↑(q) ⊆↑(p).

The logic rhymes with part (1).
# William Taysom 2019-09-25 17:48 #
Hello friends,

My father was having some difficulty with Exercise 1.66 of Seven Sketches. Thought you might get something out of my reply.

>> Exercise 1.66. Let (P, ≤) be a preorder, and recall the notion of opposite preorder from Example 1.58.

Let's do this. I'm under the weather today, so some easy proofs are just the thing. We open up all the definitions, and the proofs fall out. Exercise 1.66 is really great, a Monument Valley of Mathematics. The concepts are deep. The proofs are simple. (I have adjusted notation in your quoted bits.)

Start with the basics. The preorder (P, ≤) is a set P and relation ≤ that is

reflexive: p ≤ p for all p ∈ P, and
transitive: if p ≤ q and q ≤ r then p ≤ r for p, q, r ∈ P.

The opposite preorder op(P) := (P, ≥) flips the order of arguments so p ≥ q if q ≤ p. — Isn't ≥ the most obvious symbol?

Notice that the ≤ of (P, ≤) is implicit in op(P). Math texts like their implicitness. There is a regularity to it; however, it can be disorienting if you aren't following along perfectly. The alternative explicitness would likely overwhelm with details.

>> 1. Show that the set ↑(p) := {p' ∈ P | p ≤ p'} is an upper set, for any p ∈ P.

So ↑(p) is the set of all things bigger than p. (Non-strictly of course because saying "at least as big as" sounds terrible.)

What's an upper set?

For a preorder (P, ≤) an upper set in P is a subset U of P such that if p ∈ U and p ≤ q, then q ∈ U.

In other words, an upper set is part of the top of P. So (1) is trivial. ↑(p) is exactly the part of the top of P with p at the bottom and nothing else hanging on the sides. Let's unwind the definitions to do this formally.

Consider q ∈ ↑(p). This means p ≤ q. Take r where q ≤ r. By transitivity p ≤ r. This means r ∈ ↑(p).

This sort of rhythm makes a logician happy. What makes a mathematician happy? Considering the converse. Is every upper set ↑(p) for some p?

No! We only need two elements {a, b} for a counterexample where a and b are incomparable – neither a ≤ b nor b ≤ a. Generalize? Whenever p and q are incomparable, ↑(p) ∪ ↑(q) will be an upper set, but there won't be an r such that ↑(r) = ↑(p) ∪ ↑(q).

Maybe we could extend the definition of ↑ to sets. We would want ↑({p, q}) = ↑(p) ∪ ↑(q). So for any set S, let ↑(S) := {p ∈ P | s ≤ p for some s ∈ S}. This gives us a neat little way to characterize upper sets. U is an upper set if and only if ↑(U) = U. Nice, natural. An upper set is one closed under ↑.

Another path of inquiry... given p and q, could there sometimes be a special, best r, such that both p, q ∈ ↑(r)? What qualities would this best r have? Well if p ∈ ↑(q), then r should be equivalent to q. Moreover for some s if both p, q ∈ ↑(s), then s ∈ ↑(r). Turns our there's a name for this kind of r, it's the meet of {p, q}. So you should read Section 1.3.

One more idea... What happens when we take a totally ordered set, one in which every two elements are comparable. Then is every upper set ↑(p) for some p?

No! Consider (ℕ, ≥). All of ℕ is certainly an upper set, but ↑(n) never catches all of ℕ not for any n ∈ ℕ. How about we extend ℕ with this limit point ω <https://en.wikipedia.org/wiki/Ordinal_number>! Or try this: start with the rational numbers, fill them in somehow <https://en.wikipedia.org/wiki/Dedekind_cut>. Generalizing to preorders, we eventually discover Galois connections <https://ncatlab.org/nlab/show/MacNeille+completion>. Read Section 1.4.

I told you this book knows what it's doing.

# Bartosz Milewski 2019-06-21 15:54 #
The intuition about sets is that elements are distinguishable, you can give them names, therefore you can interchange them. A function between sets is a special kind of relation. A relation is just a set of pairs. So if you have a set {a, b}, you can have a function (set of pairs) {(a, b), (b, a)}, which maps a to b and b to a. Another function is {(a, a), (b, a)} (constant function). On the other hand, {(a, a), (a, b)} is not a function (but it is a relation). Identity is {(a, a), (b, b)}. Hope this helps.

# William Taysom 2019-06-19 19:49 #
@Drewverlee 1. Plain mathy sets. Like a set with three things in it {A, B, C} . For the sake of discussion, we will be fine sticking to finite sets. (The interesting qualities of infinite sets aren't going to be relevant.)

2. Sets are unordered. Interchange does not exactly mean changing order — but there is a connection. Remember what a function from {A, B, C} to {A, B, C} does. It tells you what A goes to, what B goes to, and what C goes to. One function is {A ↦ B, B ↦ C, C ↦ A}. Another is {A ↦ B, B ↦ A, C ↦ A}. For short, we can use order to name these functions. Let [BCA] = {A ↦ B, B ↦ C, C ↦ A} and let [BAA] = {A ↦ B, B ↦ A, C ↦ A}. In this [xyz] notation, x is what A goes to, y is what B goes to, and z is what C goes to. What's the identity function? [ABC]. Play with some compositions [BCA][BAA] = [AAB]. (Here I mean [BCA] followed by [BAA].) Do another [BCA][BCA] = [CAB]. [CAB][BCA] = [ABC].

Something extra subtle is going on here. Given an set {A, B, C} – unordered, you can use an ordered list [xyz] :: {A, B, C} → {A, B, C} to represent function on {A, B, C}. An "interchange" (or permutation) is a function which has all three elements as values. So [BCA] is a permutation and [BAA] is not. A bunch of permutations of the set closed under composition is called a "group." There are a few groups over the set with three elements, but the group of all the permutations is the fun one https://en.wikipedia.org/wiki/Dihedral_group_of_order_6.
