# Andy F 2020-05-04 10:18
Hi folks, let me do my introduction to this delightful community. I’m an industry coder currently doing webdev and frontend at Shutterfly. A few years back I was building a cool language for live programming called Circa-lang, which I showed at a few conferences. That project is no more but I’m still interested in building the next generation of dev ecosystems. These days I’m more interested in how we can expand the role of databases and/or spreadsheets, and reduce the need for coding.

On my bookshelf right now is a biography on Wittgenstein, and a Minecraft story book (since the kids are full-on crazy about Minecraft right now). Nice to meet you all

# Andy F 2020-05-04 13:24
Great question, I think the biggest mistake was trying to build a closed ecosystem from scratch, including the language, standard libraries and a new game engine from scratch. A classic bad idea! If I was redoing this project then it would definitely leverage an existing platform, maybe the web, or maybe integrate with the Unreal/Unity engines.

Also in terms of doing live programming well, something that I appreciate more now is that it’s more than just a programming language problem, liveness also needs all the build system, tooling, and runtime to be involved too.
