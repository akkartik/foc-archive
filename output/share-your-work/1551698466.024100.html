# Tudor Girba 2019-03-05 09:39
@Stathis Sideris Indeed, the one-rendering-tree was both foolish and totally worth it :smile:. We can now express interfaces that are simply not possible otherwise. The interesting thing is that we do not yet know what we can do with the new found freedom.

For example, take a look at this: https://twitter.com/feenkcom/status/1042163539950153728

In this interface, the arrows go from inside the text editor to the other editors. You can actually also type in the editors and the arrows are added by the syntax highlighting live. This is possible because the text editor is made of elements and because the graph layouts can be expressed as regular layouts. This means that we can arbitrarily combine graphs-like interfaces (and graph structures are common place in software) with typical widgets.
# Tudor Girba 2019-03-05 09:42
>Also the ad-hoc development tools idea is very interesting, especially because IDEs provide some “canned” queries on your system but in such a locked-in manner that it makes them marginally useless

We call this property moldability, and it is a central property of our environment. The interesting thing is that it exercises the way you build the system in an orthogonal manner with respect to just testing / functionality.

The other thing about it is that these tools can make the internals of systems explainable in ways that other humans can relate to very quickly. This has all sorts of implications. For example, when doing domain-driven design we only draw a picture on the whiteboard when the system does not exist. Once the system exists, it is its responsibility to draw it. This then increases greatly the ubiquitousness of the language.
