# Duncan Woods 2019-01-20 09:14
Hi, I'm Duncan,

I'm building https://thorny.io, an interactive notebook for decision making.
I want to help engineers tackle overwhelming design, planning and strategy problems where multiple decisions are tangled together.

The end goal is not a programming tool but it is still an end-user computation problem because at their most general level, decisions, plans and reasoning, are actually types of computation.

The underlying structure is a knowledge graph which includes possibilities and reasoning (similar to syntax tree).
Like a spreadsheet, if you change the reasoning, decisions or beliefs, then the consequences ripple through the graph.
The UI I have been exploring is
- an outliner (effectively a structure editor for a reasoning notation)
- with spreadsheet power (it supports formulas that reference other parts of the knowledge graph)
The demos on the site are out of date and baffling, I will update them with the current version shortly!

I am based in Bristol UK. I studied CompSci at Cambridge and was previously head of software at an engineering firm building various design and optimization tools for engineers. I am interested in the typical Brett Victor topics plus:
- non-monotonic reasoning
- belief networks
- logic programming
- argument mapping
- optimization and solvers

I have also built various compilers, code generation and visualization tools.
I want to work more in the open and share more but I find it tough.
I am so impressed with @Steve Krouse and this community, I have so much to learn here!
