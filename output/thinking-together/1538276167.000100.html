# Nick Smith 2018-10-02 03:49
@Cyrus Omar I have a question :stuck_out_tongue:. How does Hazel compare to other functional structure editors like Lamdu? Is it just about formalizing program holes, or would Hazel also lead to substantially new editor capabilities? And also, do you think that structure editors need usability improvements before they have a chance at being widely adopted? Is that part of the Hazel project?
# Cyrus Omar 2018-10-02 12:36
Hazel and Lamdu are definitely quite similar. So far, the Hazel project has been more focused on theoretical foundations and Lamdu has a much more developed UI. The Lamdu UI is much more "experimental" -- with Hazel, we're aiming for something that feels mostly like text except when text doesn't make sense.
# Cyrus Omar 2018-10-02 12:36
Hazel has a different approach for running programs with holes and live programming.
# Cyrus Omar 2018-10-02 12:37
Briefly, Lamdu stops at holes and tracks all function applications, whereas Hazel continues past holes and tracks closures only around holes. There is more on that on our draft paper at hazel.org.
# Cyrus Omar 2018-10-02 12:38
Going forward, we're definitely going to be doing more work on scaling Hazel up and experimenting with the UI. There is definitely a lot more left to do on the UI side. I think we're gonna keep exploring the "mostly text-like" ("hybrid") structure editing approach.
# Cyrus Omar 2018-10-02 12:38
I don't know if it will be practical or not long-term. I think there has been some reason to believe it can be with recent empirical work with mbeddr in particular.
