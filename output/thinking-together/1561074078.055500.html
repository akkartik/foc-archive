# Yair Chuchem 2019-06-23 02:07
@Drewverlee Yeah I guess the syntax does warrant some explanation

digits
num num // base
⇒ base

Is equivalent to Python style digits(num=num//base, base=base).

The ⇒ a b c syntax is sugar for a=a, b=b, c=c similar to JS “Enhanced object literals” i.e {a=1, b}, aka Rust’s “Field init shorthand” or Haskell’s “Field puns”.
