Glad to hear Observable is thinking about this! My solution so far has been to do Jupyter-style, except the kernel independently maintains full state of the notebook instead of just eval-ing each cell as needed. So the kernel maintains the DAG.

But I realized that what I'm doing is trying to make an imperative language work like a dataflow language, so my last few weeks have been digging up old literature on dataflow programming. One idea I have (and have started to pursue) is to implement a dataflow language with a wasm interpreter.
