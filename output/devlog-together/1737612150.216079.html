# Eli Mellen 2025-01-23 04:19
@Kartik Agaram something you may run into with the single-file HTML/JS approach to be aware of is that some devices won’t execute JS, or will only execute a subset of JS when reading the file directly from the file system as opposed to through a web server. This is a security measure to prevent folks from being able to easily embed and run malicious JS in all kinds of files (like PDFs :melting_face:)
# Tom Larkworthy 2025-01-23 05:29
it runs JS fine, its just file:// is considered its own origin so it can't do cross origin requests... e.g. fetching an external image. The security context is different than normal. There are work arounds because JS can manufacture in-memory URLs that are, by construction, in the same origin. But yeah, you are in heavy yak shaving territory at that point, and you need a good mental model of the security model to work around those limitations if you need to.
