# Konrad Hinsen 2020-10-14 10:22
You can't learn the Pharo language without the system. The language is just for methods and small code snippets. The higher-level structure of the code, starting with class definitions, is not defined by language constructs but by interacting with the system. For example, if you want to create a class Foo, you kindly ask the class Object to make a subclass of itself called Foo. Or you use a development tool like Code in GT to kindly ask on your behalf.

# Ivan Reese 2020-10-14 12:47
How much is Pharo similar to (or dissimilar from) other dialects of Smalltalk?

In other words — Is Pharo as much of a departure from other Smalltalks as GT is from Pharo?

The point of this study group, as I understand it, is to take a bunch of folks who are curious about Smalltalk and GT, and get them up to speed on both. So I do think it's worth trying to chart a path that gives us experience with Smalltalk without GT, and then Smalltalk with GT, just so we get a sense of what GT is doing for us.

If we only learn Smalltalk in the context of GT, that reminds me a bit of what happens when folks learn OO in the context of (say) Java. They think that OO means abstract factories, interfaces, Person extends Animal. They don't learn OO principles separately from a language that implements a version of them. I'd like to learn GT / Moldable principles separately from Pharo/Smalltalk, so that I can appreciate both in a broader way. That's my angle, at least — but I'm open to the group consensus.

# Tudor Girba 2020-10-14 14:45
Every Smalltalk dialect evolves separately. But, for most of them, the object model and the core libraries share a great deal of similarities. Furthermore, the development environment will share strong similarities between dialects.

Pharo is an open-source Smalltalk that has significant engineering behind it. It also has distinct language constructs like Traits, and it has a very nice reflection infrastructure (called Reflectivity).

GT reinvents the environment. not the language.

# Konrad Hinsen 2020-10-14 21:56
@Ivan Reese There's nothing specific to Pharo or even Smalltalk in the idea of moldable development. It looks straightforward (but of course an enormous implementation effort) to transpose the idea to other live coding environments, e.g. Common Lisp or JavaScript. Not sure about traditional compile-and-run languages, that looks like a harder challenge.

As for Pharo, at the level covered by the MOOC, for example, it's no different from Squeak (from which it was forked originally). You have to get rather deep into Pharo to see differences.
