If you use traditional function call syntax, f(x,y,z), then some of the consequences are:
• f((x,y,z)) passes a single argument, which is a tuple, just like Python. f(x,y,z) is something quite different: it is a function call with 3 arguments.
• f([x,y,z]) passes a single argument, which is a list, just like Python. f[x,y,z] is something quite different, probably array indexing syntax.

# Doug Moen 2020-01-31 06:33
Python: f(a, -b, c+d, g(x))
Haskell: f a (-b) (c+d) (g x)
Lisp: (f a (- b) (+ c d) (g x))
