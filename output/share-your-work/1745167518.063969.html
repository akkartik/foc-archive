# xyzzy 2025-04-23 12:05
Perhaps I could make myself a bit more clear. I would encourage lisp or smalltalk programmers to make performance, ease of use and easy to read documentation a priority.

This lisp implementation - https://macoy.me/code/macoy/cakelisp - which addresses games and tries to achieve C like performance both in terms of computation and memory is something many programming languages sorely lack not just lisp.

While we live the abundance of webapps, workflow pipeline builders, mobile apps or landing pages, the following

1. Operating systems
2. Database engines
3. Browser engines
4. Game engines
5. AV or 3D Editors
6. High performance servers
7. Embedded stuff
are all scarce. I have more respect for the C family languages precisely because of this. Bottom line, can your language do a AAA game or atleast a AA game ?
