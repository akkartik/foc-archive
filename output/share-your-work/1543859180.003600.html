# Duncan Cragg 2018-12-03 14:32
In this podcast, you describe two of your influences: "Out of the Tar Pit", and Conal Elliott's "Can functional programming be liberated from the von Neumann paradigm?"

I read them both (thanks for the tip-offs!), and both in turn refer to Backus' 1977 Turing lecture "Can Programming Be Liberated from the von Neumann Style?".

This turns out to include an early (the earliest?) description of the approach that I'm using in Onex! Again, thanks for the indirect tip-off!

Backus called the approach AST or Applicative State Transition systems. Here, you have pure functions that transform a state into a new state that replaces it.

I'm guessing it wasn't very popular, as I have never come across it until today (AFAIR), and I've been researching this whole area since the 1980s! :slightly_smiling_face:

# Gregg Tavares 2018-12-04 08:02
In listening to the latest podcast a few thoughts.

I'm sure you already know this and I know 100% it's not what you're trying to create but just in case you don't know, Microsoft Word and Microsoft Excel (and I think Microsoft Outlook, and probably Project and PowerPoint) all have Visual Basic built in with an entire giant IDE with form editors, code editors, etc... I know they aren't running a fancy language, they also are not the actually app code itself but they do let you modify the apps operations and lots of companies use them for custom solutions.

Google Docs/Sheets/etc added JavaScript scripting apparently because Office users were pushing to be able to create similar solutions to the ones they had in MS Products.

There's also things like Photoshop is scriptable from JavaScript and of course a lot of Mac software is scriptable from AppleScript. Windows software is often scriptable from any language that speaks the Windows Scripting Host protocol. That used to include Visual Basic, Perl, Python, JScript and probably includes C# and F# now a days.

Of course on both platforms every app has its own API and is often undocumented. On the Windows one there was a way for each API to expose docs as part of its public interface and there's an API explorer Microsoft provides. It's been so long since I've used it I don't remember what it's called. It might be the OLE object explorer.

That's another tech that I think died out I think. OLE (Object Linking and Embedding) was an Microsoft Standard that let you embed a view of one app inside another. You used to be able to for example select a block of cells in Excel and paste them into Word and have them linked live and if you clicked on the cells in the word document parts of the excel interface would appear. It's been so long since I've used MS Office I have no idea if that's still a thing but they had a standard and 10 to 50 apps supported it. https://en.wikipedia.org/wiki/Object_Linking_and_Embedding

Just passing on info.

I have some thoughts but will let them percolate for a while
