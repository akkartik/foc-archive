I'm puzzled that you talk about "parsing" and then mention "disambiguation". In my mental model, parsing and semantic analysis are clearly separated; parsing does not imply disambiguation. I find it is often useful to write code that is both syntactically and semantically invalid; I can imagine a structured editor that keeps the syntax tree valid without getting in the programmer's way, but it's hard to imagine achieving that for semantics.
