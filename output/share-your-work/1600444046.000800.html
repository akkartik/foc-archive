# Grigory Hatsevich 2020-09-18 08:47
I made a Chrome extension which reads websites aloud (including slack) and has, I believe, a convenient interface based on keyboard shortcuts (change tempo and pitch, pause/resume, previous / next sentence, toggle auto-scroll). Although this is not a programming tool, it might be useful for everyone who reads a lot of texts and sometimes wants to continue consuming a text while giving rest to one's eyes.

Concerning interface design, the one thing which is somewhat novel (or at least not mainstream): buttons' names on the widget contain keyboard shortcuts themselves (like "Space"), while functional description (like "pause/resume") is at a side. The motivation is to emphasize and encourage using keyboard shortcuts instead of mouse-clicking on buttons. Do you find this approach useful?

I would be happy to hear feedback of any kind. (This is my first software product which is indended for a potentially wide audience.)

Install: https://chrome.google.com/webstore/detail/read-aloud-with-au…

Demo: https://www.youtube.com/watch?v=Nnxh91ryrdU (I speak there in Russian, but the video and the widget itself are self-explanatory.)
