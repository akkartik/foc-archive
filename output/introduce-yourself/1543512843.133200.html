# Michael Arntzenius 2018-11-29 09:34
Hi! I'm Michael Arntzenius, I go by "rntz" online. I used to think I would reinvent programming all by my lonesome. Then I realized it might be a tad more difficult than I thought, so I scaled back. Nowadays I'm mostly chugging away in academia at little problems that I can make headway on, like: how do we combine functional and relational/set-oriented programming nicely? (For that, see http://www.rntz.net/datafun/) But I'm still interested in the bigger picture. Themes I keep coming back to are:

1. Small programs. Ideas last longer than code. So make your code small enough it becomes an idea.

2. Monotonicity: systems that always go in "one direction", eg. append-only databases, CRDTs, LVars, propagators, recursion in Datalog...

3. Incrementality. I think a lot of code is incidentally complex because it's trying to handle change efficiently. How can we make this easy?

4. Unexpected uses of simple type systems. Eg. To enforce monotonicity; or, as dynamic contracts rather than static guarantees; or, as data for metaprogramming with.

5. Communication. Maybe the future is in lots of special-purpose tools/langs, not one tool to rule them all. But for that to work, we need tools that work together well. Unix byte-streams were a useful first step in this direction, but are too limiting.

I joined this Slack after seeing on Twitter that there had been a meetup in London. I live in Oxford, so maybe I can come down the next time a meetup happens.

Also, hello @Thomas Ballinger!
