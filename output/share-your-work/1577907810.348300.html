# Sol Bekic 2020-01-03 02:40
The other two questions are part of mmmfs itself:
the loading behavior is very bad (or sad) due to a combination of issues that I would have liked to address during the working time but ended up not having time to. It goes something like this:
- the server and client share most of the mmmfs code and abilities, but there are some differences: only the server can compile MoonScript to Lua, and the two Markdown implementations (sadly) have minor differences. Only the client can use some JS libraries (mermaid.js, svg.js)
- the server pre-renders a static version of the page when a page is requested
- the client re-renders the page after initializing the runtime and fetching the sources. This is often necessary because of the differences above, but it currently happens in any case, because I don't know which pages need or don't need this to happen.
- fetching resources and is currently happening completely synchronously, and rendering is running in fengari.io, both of which makes everything pretty slow. Also conversions aren't cached, and some conversions involve HTTP request.
- the flash to white while the client rerenders happens a bit to early, because of the non-async implementation of that part.

So yeah, lots to improve there. You can check the server-side version my removing the +interactive from the URL end; in the case of the thesis the markdown difference ends up formatting the sidenotes in a particularly ugly fashion as you will see.
# Sol Bekic 2020-01-03 02:46
As for the facet dropdown, facets are the typed (and named) attributes of fileders storing all content in mmmfs. Sometimes they are used rather as different attributes (such as a human-friendly title for a folder), sometimes they are used as alternative variants of the same content (gallery and slideshow view of a list of pictures). For the thesis, there are two variants; one which only links to the project log and one which embeds all pages (for printing).
# Sol Bekic 2020-01-03 02:47
In the Inspector, you can view the contents of all facets of the current fileder, and see the list of child fileders. On https://sandbox.s-ol.nu you can also edit both.

The system already supports viewing the content though, just specify text/html as the content type explicitly - e.g. https://mmm.s-ol.nu/articles/mmmfs/:text/html .
