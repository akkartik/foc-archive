Of course, this is complicated by the fact that it's been a trend among programming languages to have utterly unholy build processes, whose brittleness make it hard to package things‚Äîbut if you can't build the software in question, you can't patch it, and you probably shouldn't be using it in a container either. This issue isn't really the fault of any system in particular but rather of programmer trends, with programs like web browsers setting trends (and records) with their compilation resource usage and dependency chains.

# S.M Mukarram Nainar 2020-07-13 11:55
While static builds to simplify some things, they don't solve the versioning problem‚Äîthey just let you keep using software that's out of date, which you don't want to be doing anyway. Package managers fix this issue since good ones will track library dependencies and rebuild as necessary. I'm not sure what to blame for build system bloat really, it seems more of a cultural issue than anything‚ÄîI understand that google has buildfarms for stuff like chromium so the core devs don't have to worry about it, I generally have trouble reproducing builds for any sufficiently complicated c++ program, and I'm not too sure why.
