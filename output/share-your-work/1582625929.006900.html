# Martin Sosic 2020-02-25 02:18
Feedback request: brother and I just finished first prototype of Wasp - higher-level programming language for building web apps - would love to get your feedback!
Webpage (with demo): https://wasp-lang.dev, Github repo: https://github.com/wasp-lang/wasp .
We are still very early / experimental, and current demo is yet far away from our vision, but it should illustrate some basic concepts.
Our main challenge is designing the language so it captures the correct abstractions and is easy to use, while also making it flexible enough to be useful. Plan for now is to find inspiration in current popular frameworks and similar solutions + guide development by specific use cases (we started with Todo App).
Thanks!

# Martin Sosic 2020-02-26 05:26
Thanks @breck7!

> long term advantages
Long term advantage would be code that is closer to specification, shorter and more maintainable. Also, easier to write, lowering the knowledge barier needed to build a web app and allowing faster prototyping.

> you really need a number of DSLS
I am not sure what exactly you mean by this, so pls correct me if I did not answer the question.
Idea of Wasp is that you get a lot of stuff done in a single DSL, that spans the concept of the whole web app. However, if there is something you need to customize in more details, and there is a good way out there to do it that people love, it probably makes more sense to do it that way then to reinvent the wheel in Wasp. Example of that is describing styling -> css is made for that and although it has its issues, people use it, and it would be too big task for us to reinvent it, so Wasp will not try to replace css, but instead allow using it. Same goes for html/jsx for layouting, React/Vue for reusable components, Javascript/TS for general / business logic.
Wasp as DSL serves the purpose of gluing this all together, removing unneeded complexity and hiding ugly/boring parts.

> language tooling
Good question! Since we are imagining it as a pretty horizontal DSL, I believe it will be very important to have a good way to discover options, so some kind of auto completion could help a lot. With LSP I am imagining this should not be so hard to develop for a couple of main editors, but I have never been developing language tooling so it is all a guess :).

I hope these answers are addressing the questions well - I would love to hear your opinions on these topics!
