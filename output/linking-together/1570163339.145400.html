# Daniel Garcia 2019-10-03 21:28
I was reading the description of Tic-tac-toe in Wikipedia and start daydreaming on the smallest and easier to understand program for it.
First here is the relevant paragraph from Wikipedia: The game can be generalized to an m,n,k-game in which two players alternate placing stones of their own color on an m×n board, with the goal of getting k of their own color in a row. Tic-tac-toe is the (3,3,3)-game.

I came up with a program divided in 3 parts:
1. A declarative set of rules
2. A state machine for game transitions
3. A visual specification for the screens

Here are the set of rules:


Space: X, O or empty
Board: 3x3 table of spaces
Board full: Board has no empty spaces
End game: Board full or Win
Win: Row, Column or Diagonal of same Player token
Draw: Board full and not win

Player token: X, O
Human player: Choose a player token
Computer player: Remaining token not chosen by human player

Player move: Choose empty space (Choose could be done with the mouse, keyboard, or other input device)
Computer move: Randomly choose empty space


Some rules in a Haskell like language:

Space = X | O | Empty
Board = Space[3][3]
fullBoard board = none board (\space -> space == Empty)


I tried to have the rules read as close as possible to English. I see some rules really close (and easy to represent) in a language like Haskell.
Other rules are probably to high level to be easy for a computer to understand (specially the winning logic)
