# nicolas decoster 2021-07-26 08:35
Wow! Really really cool ideas here.

What I like:
• Use of 2D to simply organize your work. In the end, your work is in some kind of map, that you have build and that you can visually remember.
• Use of links only for event triggers.
• In fact it is kind of similar to traditional nodes'n wire environments except as there is no text by default in nodes, you can look to the patch from any angle, and it is the point of view of the user that defines the text orientation when you need to display it.
• To increase the volume parameter for a node you drag'n drop new "volume+" cells on the node and the volume will be higher if there are more cells. Same with other parameters.
