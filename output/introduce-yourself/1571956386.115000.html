# Edward de Jong 2019-10-25 00:42
software is malleable like sand is malleable. Software has a very tiny cell size, like sand, and as such is very tedious to construct anything. It is not really a flexible material, it is merely a microscopic rigid material, and has to be used in great abundance to achieve good things. The fact that we are not in an era of interchangeable parts means that people have to build their castles over and over custom made, instead of using pre-fab small pieces like lego. Lego is arguably the greatest toy ever invented, founded on the 8 bump brick which snaps together in many ways, but always perfectly. We need better data interchange between pieces and then it will be a much more flexible medium. It's not locked down really; none of the pieces snap together that well.
# Dan Cook 2019-10-25 08:14

Anyway, nice analogy. Software is super powerful and expressive, but we need to find ways to make it less like working with sand. The tricky part is that we've got to use sand (at least initially) to shape the sand into something less sandy.

# Dafydd Rees 2019-11-20 05:41
Power is one of the key problems with software. It’s not just a problem with the iThings and the appstore. It’s also a problem perennially present in Smalltalk where any code can do anything, and it’s more subtly present in the teamwork dynamics of software development. What developers would you trust? What software libraries and applications do you trust? The people that write the code (especially in languages like smalltalk and in compilers) are the people that “twiddle” the DNA of what we’re allowed to do… ?
# Dafydd Rees 2019-11-20 05:43
In one bank, the lead dev pretty much didn’t trust us to make decisions - he wanted to monopolise them all. We ended up having secret meetings in the building to be able to talk openly about how to build things. That’s just an extreme example of the problems associated with “power” in software. Scaling software may be at least partly difficult because of the need to handle trust/power at scale.
