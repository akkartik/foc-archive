# Sol Bekic 2019-09-26 03:38
I'm building a typed file-system to replace application-centrism in the spirit of end-user-programming.

A lot of ideas are jumbled up together in this project, which makes it a bit hard to talk about, but the core of it is that content is stored in typed 'facets', and the system has a built-in (aspirationally user-extensible) library of known transformations it can apply to these; such as for example rendering markdown to DOM, executing Lua or JavaScript, thumbnailing pictures, loading remote content via links etc. These transformations are automatically combined and searched; whenever content is displayed or embedded, the code embedding the content can ask for a target format/constraint set and the system takes care of e.g. fetching a remote script file, executing it to obtain a markdown string, then rendering that string to a DOM element which can be included (this example would have the type string URL -> application/javascript -> text/markdown).

Another big part is that these facets are attached not just to leaf nodes in the tree (conventional files) but also to the folders. This means that any container can be scripted, for examle a 'pictures' folder can be built to show either a grid view or a slideshow of images it contains, or an UI to switch between both, and the images inside can be images stored on disk or links to images on the network, and the folder doesn't need to care.

The system has been up and running as my website for a while now, running either server side (static html output) or clientside (live DOM output) but with no ability to edit content. For my Bachelor Work I am now planning to add editing capabilities in the front-end, that should also be scriptable similar to how the output has been scriptable so far - the idea being that this would enable the creation of e.g. a document/folder behaving like an MS Word document, containing images, videos and text, but being editable in page layout; or for example a pinwall/moodboard for arranging snippets on a pannable canvas etc.

If anyone is interested to see some examples, the website is https://s-ol.nu and comes with a little inspector to look at the system contents. The code for everything is also public here: https://git.s-ol.nu/mmm/

If this rings any bells with anyone, it would be very helpful if you could point me to any theory that I could relate to in my thesis. Thanks!

# Tudor Girba 2019-09-27 10:03
@Steve Dekorte We use the term moldable development. Now, it is important to note that through moldable development, views are constructed by the human for the human. It is certainly conceivable to generate them automatically, too, but the intent is the key. We literally build hundreds of such views for every system we work on. For example, a stock GT ships with some 500 views. And in a typical system we would have some extra hundreds.

Like others build tests, we build a whole environment, and weave narratives about the internals of systems into live documents, too.

It is also worthwhile to note that the cost of creating these views is measured in minutes (literally). The average is usually around 10 lines of code.

It is also interesting to note that the concept of moldability applies throughout the whole development. For example, we also have custom searches. That is, an object knows both how to present itself and how to search itself. And the same applies for debuggers or coders.

# Tudor Girba 2019-09-27 10:50
We first want to reimagine the existing development experience. Our aim is to make systems explainable.

We are already working with non-technical people in real-life projects and it turns out that, while they do not necessarily want to create views, they certainly want to consume them.

At the same time, if we look at something like Excel, there is quite some evidence that regular humans can create views, too.

The other thing to note is that the work around moldable development is not just a hypothesis. It is a hypothesis tested empirically over one decade in real life systems. For example, Pharo ships with the first incarnation of GT since 5 years.
