Hello! I am a frontend developer from Kharkiv (Ukraine). I contribute to open source projects and work on my own framework called Rete.js. http://rete.js.org

I started learning programming more than 5 years ago, studied languages ​​ PHP -> JS -> C++ -> C# -> Java -> ES6/TS. Now basically I programming on JavaScript.

At the moment I see the benefit for myself in languages ​​such as JS/TS and C++, but one of the problems still remains - writing the code. Perhaps someone will argue that you need to have a good knowledge of programming language. I agree, good knowledge will allow you to write quality code, but there are problems with reading someone else's code that is written in a different style (or has poor quality).

Being inspired by UE4, Substance Designer and Plant factory I decided to write my own library for creating such editors in the browser (dataflow programming). At that time (as now) I didn’t find a universal and customizable framework/library. I have started writing a library called D3-Node-Editor (March, 2017) and after a year of development I revised many approaches and rewrote it to the new architecture and called Rete.js. https://github.com/retejs/rete

Are there somebody who have experience in Dataflow programming and know the current unsolved problems?
