# Daniel Worthington 2020-12-10 10:46
:wave: Hello! I'm Daniel, and my passion is for design tools. Specifically, I’m very interested in tools that allow designers to have more autonomy, allowing them to ship things and freeing them from relying on engineers. I’m a designer and engineer myself, currently working at InVision as an engineer on the Design System Manager team.

My big project at the moment is to articulate the vision I have been working on for the last few years in a meaningful way. It’s all about making it possible for designers to build user interfaces and collaborate with engineers rather than draw pictures of things and then “hand off” to engineers. Realizing this vision would involve creating several new tools, and — maybe more important — a new way of thinking about what designers are responsible for and how they collaborate with engineers.

# Daniel Worthington 2020-12-10 11:09
Yes, Sketch.systems is an interesting project. I was also closely following Subform (made by the same people) while it was in development. Part of what I think is needed is the ability for both designers and developers to work in a sort of “UI only” mode where the business logic is simulated (faked really) and the data is pre-built samples. There is a lot of power in being able to easily put your UI into an interesting state or to simulate error conditions, empty states, slow network calls, etc. Or to just be free of having to set up and run all that code! I think a tool that lets you simulate the logic of an application for those purposes might look something like Sketch.systems
