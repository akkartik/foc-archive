# Edward de Jong 2020-01-16 14:47
@Mariano Guerra as for Frink, it is quite an unusual beast. Frink is what i would call a kitchen-sink kind of system; it offers language translation, units conversion, a programming language, a java servlet based server side programming that resembles PHP in that you interleave your Frink language with regular HTML. It is a very complex product, and is about 20 years old. Both Frink and Beads store physical units of measurement as an array of fundamental units that is tracked as arithmetic proceeds, so that length * length becomes an area. Frink covers more esoteric units than Beads does (such as Assyrian cubits), and he has done a terrific job of identifying the flaws in some of the standards.
