# Tudor Girba 2021-09-14 22:34
GT comes with a new graphical stack and a new environment that is distinct from the one of Pharo, even if the first generation of GT was incorporated into Pharo. For example, in GT, the Playground is a dedicated Lepiter Pharo snippet.

I am not sure why you say that it does support multiple lines. Could you detail?

About the tutorial you just mentioned, this will exercise Morphic, the classic graphical framework from Squeak/Pharo. In GT, we rely on our own which is called Bloc, like in the code you saw in BlBasicExamples. That class contains example methods (unary methods annotated with <gtExample>). When you see such an example method, method coder will offer buttons to execute and inspect the result.
# Tudor Girba 2021-09-14 22:38
You get "You see a static preview because this element already has a parent." when you are inspecting an element that already exists in a scene.

For example, inspect this snippet:
container := BlElement new.
element := BlElement new 
	background: Color red.
container addChild: element.
element

Here, we add an element in container and then try to inspect the element. The Live view will try to display that element by placing it into under the inspector rendering tree, but as the element already has a parent, it will show only a picture of it instead of the live element.
