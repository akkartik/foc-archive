# Owen Campbell-Moore 2021-11-02 12:03
Hello Friends! I am Owen, an ex-Google/ex-Facebook person working in SF on the future of coding.

My background / journey in this space:
• I worked on Google Chrome for 3 years, leading Progressive Web Apps and the future of x-platform development
• I concluded it was possible to build amazing web apps, but far far too hard
• I am disappointed nobody has solved “write once, run everywhere”, but where the output is UI native to the target SDK (and includes Web Support), and adapted to the OS / form factor of the device 
• I became very interested in React Native development and love following the advancements in the React space
My conclusions:
• I believe we need to start coding at a way higher level of abstraction, i.e. describing the features of our products at a semantic level rather than UI and in terms of complex low-level logic
• If we can increase the level of abstraction for coding for certain classes of apps I believe we can enable:
◦ “Write once run everywhere”, but where the high level descriptions of products can be compiled into super polished native UIs best suited to the form factor of the target device.
◦ Making it 1000X easier for people to build apps and web apps, since describing a feature semantically is far easier than at an implementation level
My vision for the future of coding consumer software:
• I want people to be able to re-build products like AirBnB by simply describing (in a formal Domain Specific Language, via a slick web GUI) the things users can do in the app, and then for the platform to compile this down to super polished UIs for every OS, plus auto generate a production backend.
