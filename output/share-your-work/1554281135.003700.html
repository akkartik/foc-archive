# Mariano Guerra 2019-04-03 12:18

# Mariano Guerra 2019-04-03 12:20
a long way to say that they care a lot about "configuration management" we provide a "chain" of dictionaries and lookup, there are dictionaries at the user level, then per dashboard, then per visualization, if you lookup key k1 in dict d1, it will look first in the visualization, then the dashboard, then the user, so you can start global and then specialize if needed per dashboard or visualization
# Mariano Guerra 2019-04-03 12:21

# Mariano Guerra 2019-04-03 14:25
the main difference is kind of fundamental, your tool understand types, so a collection has a type for the items, then you can show a "schema" of the item to operate on it as an explicit way to say "over all values of this shape", in my case since the inputs are schemaless the most I can do is infer the schema but I haven't done that yet, that's why I have the explicit iterator value, which is better than explicit imperative loops but next time I may go with "iteration by default, single item is a special case", something I've been exploring a little with the idea of an path item that represents a "column", that is, all items with that key in a colection, the column explicitly applies to all items and if used in a value context will work as a generator for that column (will generate a list of values for that field in the collection)

# William Taysom 2019-04-03 17:23
Big observation: I've played with some cool ideas with a notebook style for these kinds of operations. It can work really well. One view is to show data, operation, data, operation. It's like a command-line transcript, but structured rather than text. The other is more like a traditional GUI where you manipulate an object in place. You can imagine this as stacking transformations. Then switching between these views is just a matter of rotation. From there you can go wild. One natural path is to allow for branching and merging. (In the context of tables, merging nodes often correspond to joins.) If you aren't careful you find yourself in parametric modeling node-and-edges land. I don't know if madness is the final destination, but at least you end up with a kind space of interaction modalities in which familiar conventions are projections.
