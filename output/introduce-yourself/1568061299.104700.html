# Scott Werner 2019-09-10 09:34
Thanks, yeah I’m hoping to have something to interact with and share before the end of the month, . But if you’ve ever used the fugitive.vim’s Glog command to move forward and back through a file’s history, I think its possible to break that apart and have interactive visualizations where you can move forward and back at the project/file level, the class level, the function level and maybe even find ways to drop into an interactive session for any of those pieces. One idea I have is using code coverage tools to get a list of tests that touch a specific function, and then being able to start a debugger session at a particular spot at a particular point in time in the project’s history with the test taking care of all the setup for you.

Essentially all the information you need is there in the git history of the project, and I think if you combine it with a few different tools and focus your interface/UX to the use case of reading/understanding code, there’s something there :slightly_smiling_face:
