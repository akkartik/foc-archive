# Nick Winter 2019-12-12 16:56
Hi y’all, Nick here, founder of CodeCombat–we make a game-based learning platform for teaching kids coding. Got our own code engine to try to make it easier for beginners to learn languages like Python, JavaScript, C++, etc. with friendly error messages, control over code execution, full interpreter/game engine integration, a (disabled) time travel debugger, etc. Love hearing about ideas for how to make learning coding more intuitive.

# Nick Winter 2019-12-13 13:42
We did many iterations where the main model was freemium B2C web subscriptions, then built a whole teaching version for use in the classroom.

We haven’t published a huge amount about it, no. The interpreter is open source: https://github.com/codecombat/esper.js (along with plugins for various languages’ runtimes and parsers)

The frontend is also open source: https://github.com/codecombat/codecombat

A lot of crazy code lives in our game engine, which lives in the database so we can do livecoding in our level editor.

Some day I might get around to writing more of it up…

# Nick Winter 2019-12-17 11:12
@nicolas decoster I find that when you have small numbers of kids and an expert facilitator (such as yourself), the creative coding is the way to go; we are more useful in the space when a kid can’t get expert personalized instruction all the time and needs the computer system to be able to do that, where it helps to have short levels with more-or-less set tasks to aid the computer system in guiding students. That said, we’re adding more and more creative coding projects in our new classroom product, Ozaria.
