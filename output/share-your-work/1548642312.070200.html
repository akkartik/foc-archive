# Gregg Tavares 2019-01-28 19:30
> React proves that it's not inherent in the model that it has to be inefficient

i'm not sure what this means. I feel like if you tried to translate any of the complex UIs shown in this thread (https://github.com/ocornut/imgui/issues/2265) to react you'd find you have to do a ton of extra work to keep perf up in either ReactDOM or ReactNative. In fact I think React could not do it (not keep 60fps)

That's not React's fault really. It's that React is layered on top of slow retained mode GUIs libraries where 1000s of stateful UI objects have to be managed so it does the best it can by using a virtual dom but ultimately can't deliver once the UI gets complex.

If we could fix the underlying slow GUI libraries to stateless zero object ImGUI libraries then React might shine even more.
