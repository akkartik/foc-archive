# Edward de Jong 2020-06-17 01:58
@Peter Damoc There are several next gen language projects that are aimed directly at cross-platform graphical interactive software development. The "Red" language (a rebirth of Rebol) is aiming for a very wide range, and my Beads language does Web, Mac OSX, Windows, iOS and Android. Not sure what you mean by 3 big OS'es, because there are really 5 economically important programming OSes nowadays, each with millions of users. I am leaving out internet of things as it is not usually graphical/interactive, and ignoring Linux desktop because Android is 100x larger than Linux desktop economically and due to the fascistic "software IP must be free", there is no appreciable market for Linux desktop software. It is quite unfortunate that Stallman and his ilk so beat down vendors of linux products that they drove them off to greener pastures. There is also a lamentable lack of standardization on Linux executable formats and graphical layers that greatly hindered a Linux ecosystem from developing commercially. For my own project i am using Node.JS or Deno to run JS for Linux server side code. Sure would be nice if Google would just let Android take over the Linux desktop market, and make things uniform.

There is also dart which Originally landed with a thud, But the flutter cross platform graphics library has so much manpower in it that it is now quite usable. Flutter is unique in that it renders to bitmap and then transfers a bitmap to the underlying operating system. In the case of beer is it operates as a transpiler to JavaScript, So in some sense it is a direct competitor of typescript
