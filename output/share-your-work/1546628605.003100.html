# Scott Anderson 2019-01-05 18:16
I guess you could have a system where programmers link comments to code blocks and automatically flags\warns about comments that haven't been approved and\or updated for edited code. Might be a good feature for a code review system. At some point unless you have an AI in the compiler that can fully understand comments and flag comments as potentially\wrong or out of date there is going to be no perfect system. An extreme version would be write comments in some higher level psuedo code that can actually be executed and match it with the runtime version of the code. I think that would only be useful in certain niche situations though. In games\game engines that isn't totally uncommon, a lot of the time you have a slower, C++ version of a function as a reference and a optimized version using assembly or SIMD intrinsics that is harder to read.
