# Ruben Nic 2018-11-07 12:11
Hello everyone. :wave:

I want to reinvent programming, or help anyone else who is, because I fear that we are squandering human achievement otherwise. Computers are the pinnacle of human invention, a universal machine, and we have locked their use behind a techno priesthood who speak arcane runes. Instead of computers becoming the bicycle of the mind we have created TV 2.0. It seems ridiculous and hugely inefficient to me that the people with domain specific knowledge need to hire others to encode that knowledge into a computer program. We should create tools to enable the domain expert.

I’m currently working on a bimodal visualization of React Components. Bimodal because both the visual and the textual views are first class meaning changes are propagated to the other. I’m very interested in bimodal types of programming as I feel that while text is great at expressing detail, it isn’t as effective as visuals in communicating structure and relationships. I’m limiting this tool to React Components because I’m unsure if this tool will actually be very effective and I want a well defined and small space to play in.

If anyone wants to grab a beer/coffee/tea in Pittsburgh, PA and talk programming just DM me.

# David Piepgrass 2019-02-24 21:08
While it seems ridiculous that people with domain specific knowledge need to hire others to encode that knowledge, I think it will be necessary to offer some universal education to future children - not so much the traditional stuff like the syntax of Java and how to run javac, but "computational thinking", to think a bit more logically and to learn things that are more computer friendly (e.g. the line formula is commonly taught as y = mx + b, but ax + by + c = 0 generally works better in software. Bonus points if you know why.)

I am a bit surprised, though, that nothing good enough to universally replace spreadsheets has caught on yet.

Me, I want programming to be reinvented, but have decided that yes, I have a bunch of seemingly unique ideas I want to contribute, but I'd prefer to work mainly on core foundations - interop and lower-level stuff - and leave some of the reputedly more fun work, like visual programming interfaces, to others.
# David Piepgrass 2019-02-24 21:37
"Web Console is an OS X application for creating and running custom plugins that wrap command line programs in web user interfaces."

Interesting, I have been thinking that I wanted my programming language's standard library to automatically create a UI for choosing input to the program. For example, rather than write a --help screen that says

Options:
--compress:F Compresses the file F

you would just express options as optional named arguments to "main" with display-attributes attached to them, to provide the label/tooltop to show in the HTML UI, like

.entry main(
@label("File to compress:")
compress: opt!filename)
{
if (compress) {
// TODO: whatever it was we
// planned to do with the file
}
}

Then the HTML UI says

:white_large_square: File to compress: [* ▾] [Browse]

  • this is an empty combo box with dropdown history of the user's previous choices

    You could of course still use the app as a command-line app, and you could still write --help to be told about the --compress option, but the handling of --help would be auto-generated based on the attributes on the parameters to the entry point.

    My system would also show output more visually, and support multiple entry points in a single app because why should I have to create a separate app for every entry point? That's inconvenient!

    Wrapping arbitrary cmd-line apps sounds to me like a harder problem than building the feature into the programming language / standard library... but I don't really understand what it does just by looking at the gif in the readme.
