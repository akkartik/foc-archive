# Pine Wu 2019-08-25 21:06
The downside is that it's based on TextMate's grammar syntax. And as a result very hard to get right.

We are working on semantic syntax highlighting in VS Code and TypeScript is adding highlighting support. Once this becomes generally available, you should be able to use anything to highlight code, including tree-sitter based solutions (check out https://marketplace.visualstudio.com/items?itemName=georgewf…) or home-grown solutions.

# David Piepgrass 2019-12-07 09:54
The worst part about doing a textmate grammar is that there isn't any documentation AFAICT. The microsoft page says "You can find a good introduction to TextMate grammars here" but the link points to a page that basically assumes you already know approximately how to write a TextMate grammar. It doesn't explain why there are both xml and json formats, or what the basic layout of a grammar file is, or what the top-level fields like "fileTypes" and "keyEquivalent" and "patterns" and "repository" mean...

Amazing how everybody uses these grammars but nobody has ever tried to document them. (Actually I have repeatedly been baffled throughout my life at how everyone but me seems to so easily tolerate and muddle through undocumented things...)
