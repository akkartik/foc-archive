Hey, I'm working on smallweb, a self-hostable personal cloud.

It's inspired by by serveless platforms like val.town and old standards like cgi-bin. I still struggle to describe it. It feels like coding in dropbox, but in a good way.

For example, if I want to host my new blog in smallweb, I can do it by just creating a new folder at ~/smallweb/blog/main.ts :

import { Smallblog } from "jsr:@tayzendev/smallblog@1.1.5";

export default new Smallblog({
  siteTitle: "Pomdtr's Blog",
  defaultAuthors: ["pomdtr"],
});

Then, no need to run any terminal command or to wait for a deploy step, my blog is instantly available on the blog.pomdtr.me subdomain: https://blog.pomdtr.me (this one is hosted on my raspberrypi).

Apps also have write access to their own dir. For example, if you go to https://vscode-demo.smallweb.run, you'll be able to write in my own smallweb dir. Or feel free to draw me something at https://excalidraw.smallweb.run !

All of the websites hosted under the smallweb.run domain are open-source, and can be found at: https://gh.smallweb.run
