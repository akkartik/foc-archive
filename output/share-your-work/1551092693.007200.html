# Ville Ilkkala 2019-02-25 08:09
intersting, first time encountering this. Thanks for the tip! The ValOS runs the same code on node.js on servers, too, with a somewhat unorthodox approach. We have a virtual DOM running in a pretty much identical library as the front end and the back end reacts to data changes.

You can just specify a lens for the node.js server by pointing WORKER_LENS to a lens media. If the lens has an observable value in a function call, something like

<ValaaScope focus={focus.operationToDoInTheBackEnd(focus.valueToObserve)} />

then the back end Gateway will react to it, call the function and do stuff with it, which can for example write back to the same partition a new value, to which the front end can again react.
